Вся эта лабуда выполняет 3 функции:
1. Парсинг новостей из списка каналов и постинг это в собсвтенные каналы(нонстоп парсит и раз в три часа выгружает в каналы)
2. Копирование из собственных каналов в группу с обсуждениями
3. Генерация предложений по новостям для зампредов (каждому собственный регион)

Чтобы всё это заработало нужно запустить все три эти модуля (предварительно нужно активировать venv (venv/Scripts/activate))
Всё запускается из корня проекта!
1. Основной который парсит новости находится в корне проекта и запускается просто через 
	python main.py
2. Копирование новостей из каналов в группу
	python -m dvbnewsbot.main
3. Генерация предложений
	python -m newsendingbot.main

2 и 3 модули вроде не отваливаются, в целом, если они запущены, то будут работать
А вот основной модуль, может отвалится и выпасть в ошибку например если инет пропадёт, кароче за ним нужно следить периодически



ЧТО ВОЗМОЖНО НАДО БУДЕТ МЕНЯТЬ

1. Если скажут добавить какой-то канал в парсинг:
файл config/config.json
просто добавить в source_channels нужный канал
2. Если скажут поменять частоту постов:
файл config/config.json
просто заменить в batch_interval_minutes значение на нужное (в минутах)
3. Если скажут заменить получателей рекомендаций(щас это мы, возможно скажут реально зампредов поставить):
файл newsendingbot/main.py
заменить значения в REGION_USERS (я думаю там всё понятно, если че пиши) + если промт скажут поменять, то он тоже в этом main файле
4. Супер врятли, но может надо будет менять LLMку или зарегать новый api ключ для мистрали, то его менять в файлах 
config/llm_config.json и newsendingbot/llm_config.json


ДОП ИНФА

Все новые новости этих модулей сохраняются в logs/queue_for_distribution.json и newsendingbot/queue_for_distribution_recommendations.json соответсвенно
